## 1. 产品概述

### 1.1 产品名称

**Mindful** — 冥想社区平台

### 1.2 愿景

打造一个温暖、专注的冥想爱好者社区，让每个人都能找到适合自己的冥想方式，记录成长轨迹，并与志同道合的人连接。

### 1.3 目标用户

| 用户类型 | 特征 | 核心需求 |
|---------|------|---------|
| 冥想初学者 | 对冥想感兴趣，不知从何开始 | 引导工具、学习资源、入门课程 |
| 有经验的冥想者 | 已有冥想习惯，想要深化练习 | 记录工具、数据分析、进阶内容 |
| 社区参与者 | 希望与他人连接、互相激励 | 日记分享、共同冥想、许愿树 |

### 1.4 核心价值主张

1. **工具丰富** — 9种冥想辅助工具覆盖呼吸、专注、放松、引导、视觉化等
2. **记录成长** — 结构化日记 + AI 智能总结，看到自己的进步
3. **知识体系** — 从入门到进阶的完整学习路径
4. **社区温暖** — 分享日记、共同冥想、许愿墙，不孤单地修行

### 1.5 技术选型

| 技术 | 选择 | 说明 |
|-----|------|------|
| 前端框架 | React 19 + TypeScript | 已有，保持不变 |
| 构建工具 | Vite 7 | 已有，保持不变 |
| 样式 | Tailwind CSS 4 | 已有，保持不变 |
| 国际化 | react-i18next | 已有，保持不变 |
| 音频 | Howler.js | 已有，保持不变 |
| 后端/认证 | Firebase (Auth + Firestore + Cloud Functions + Realtime DB) | **新增** |
| AI 服务 | OpenAI API / 通义千问（多模型支持，用户自带 Key） | **新增** |
| 部署 | Vercel 或 Firebase Hosting | 待定 |

## 2. 功能模块详述

### 2.1 模块一：冥想工具集（9种）

#### 2.1.1 已有工具（6种，优化升级）

**① 呼吸练习 (BreathingTool)**
- 3种模式：腹式呼吸、4-7-8 呼吸、方块呼吸
- 可视化圆形动画引导呼吸节奏
- 计时器控制，可自定义练习时长
- 练习完成后提示记录日记

**② 专注训练 (FocusTool)**
- 注意力焦点可视化（中心点 + 引导动画）
- 倒计时计时器
- 分散注意力时的温和提醒

**③ 放松练习 (RelaxationTool)**
- 渐进式肌肉放松引导
- 文字提示 + 可选语音
- 分步骤引导

**④ 引导冥想 (MeditationTool)**
- 预设冥想课程（正念、慈悲心等）
- 配合语音引导
- 多种时长选择

**⑤ 自由计时器 (TimerTool)**
- 预设时长：5/10/15/20/30 分钟
- 自定义时长支持
- 完成时的庆祝动画
- 开始/暂停/重置控制

**⑥ 环境音混音器 (AmbientTool)**
- 多轨环境音叠加（雨声、鸟鸣、溪流等）
- 每轨独立音量控制
- 可与其他工具组合使用

#### 2.1.2 新增工具（3种）

**⑦ 身体扫描引导 (BodyScanTool)** `新增`

- **核心功能：** 分部位引导的身体扫描冥想
- **扫描顺序：** 头部 → 肩颈 → 手臂/双手 → 胸部 → 腹部 → 腿部 → 双脚
- **视觉引导：** 人体轮廓 SVG，当前扫描部位高亮发光
- **文字提示：** 每个部位配合引导语，如 "感受你的肩膀，注意是否有紧绷..."
- **可选语音：** 支持开启/关闭语音引导
- **3种速度：**
  - 快速扫描：5 分钟（每部位约 40 秒）
  - 标准扫描：15 分钟（每部位约 2 分钟）
  - 深度扫描：25 分钟（每部位约 3.5 分钟）
- **完成动画：** 全身亮起柔和光芒

**⑧ 冥想音乐库 (MusicTool)** `新增`

- **核心功能：** 专门的冥想背景音乐播放器
- **分类：**
  - 自然音乐（森林、海浪、雨林）
  - 器乐（钢琴、长笛、竖琴）
  - 颂钵/冥想钟
  - 白噪音（粉噪音、棕噪音、风扇声）
  - 脑波音乐（Alpha 波、Theta 波）
- **播放控制：**
  - 渐入渐出（可设置 5/10/15 秒）
  - 循环播放
  - 音量调节
- **组合使用：** 可与计时器、引导冥想等工具叠加
- **收藏功能：** 标记喜欢的音乐，快速访问

**⑨ 可视化冥想引导 (VisualizationTool)** `新增`

- **核心功能：** 动画化的视觉冥想辅助
- **3种视觉场景：**
  - **曼陀罗旋转：** 几何曼陀罗图案缓慢旋转，颜色渐变，帮助集中注意力
  - **星空呼吸：** 星空背景，星星随呼吸节奏扩散/收缩，配合粒子效果
  - **水波涟漪：** 水面效果，点击/呼吸产生涟漪，逐渐归于平静
- **全屏沉浸模式：** 隐藏所有 UI，仅显示视觉场景
- **节奏同步：** 动画节奏可与呼吸频率联动
- **引导文字：** 可选的淡入淡出引导文字叠加在画面上
- **技术：** Canvas / WebGL 渲染，保证动画流畅

### 2.2 模块二：冥想日记系统 `新增`

#### 2.2.1 日记记录表单

冥想结束后自动弹出，也可手动从日记页面创建。

**自动填充字段：**
- 日期时间
- 冥想类型（从使用的工具自动关联）
- 冥想时长（自动记录）

**用户填写字段：**

| 字段 | 类型 | 说明 |
|------|------|------|
| 情绪评分 | 1-5 级 | 自定义天气风格图标：暴风雨 → 阴天 → 多云 → 晴朗 → 彩虹阳光 |
| 专注度评分 | 1-5 级 | 自定义图标表示专注程度 |
| 身体感受 | 多选标签 | 见标签列表 |
| 心理状态 | 多选标签 | 见标签列表 |
| 自由文字 | 文本框 | 可选，500 字以内 |

**身体感受标签（15个）：**
放松、轻盈、沉重、疼痛、温暖、麻木、活力、柔软、紧绷、流畅、酸胀、清凉、刺痛、舒展、通透

**心理状态标签（15个）：**
平静、焦虑、喜悦、困倦、专注、分散、感恩、好奇、释怀、敏锐、迷茫、笃定、慈悲、自由、宁静

#### 2.2.2 日记查看与管理

**GitHub 热力图：**
- 类似 GitHub contribution graph 的全年视图
- 用颜色深浅表示每日练习时长：无色 → 浅绿 → 中绿 → 深绿 → 墨绿
- 鼠标悬停显示具体日期和时长
- 一目了然全年练习分布和坚持程度

**情绪日历：**
- 月历视图
- 每个日期格子显示当天的情绪图标（天气风格图标）
- 有多次记录的日期显示主要情绪（出现次数最多的）
- 快速浏览一个月的情绪波动
- 可切换月份

**日记列表：**
- 时间线形式展示
- 每条显示：日期、情绪图标、冥想类型、时长、文字摘要
- 筛选功能：按情绪、按工具类型、按日期范围
- 搜索：支持全文搜索日记内容

**单篇日记详情页：**
- 完整展示所有字段
- 编辑/删除功能
- 分享到社区开关

#### 2.2.3 数据存储

**Firebase Firestore 数据结构：**

```
users/{userId}/journals/{journalId}
  - date: Timestamp
  - toolType: string
  - duration: number (seconds)
  - moodScore: number (1-5)
  - focusScore: number (1-5)
  - bodyTags: string[]
  - mindTags: string[]
  - notes: string
  - isPublic: boolean
  - isAnonymous: boolean
  - createdAt: Timestamp
  - updatedAt: Timestamp
```

### 2.3 模块三：AI 智能总结 `新增`

#### 2.3.1 总结类型

| 类型 | 触发方式 | 数据范围 |
|------|---------|---------|
| 周报 | 每周日自动生成 + 手动触发 | 过去 7 天 |
| 月报 | 每月 1 号自动生成 + 手动触发 | 过去 30 天 |
| 自定义 | 手动选择日期范围 | 任意时间段 |

#### 2.3.2 总结内容

**统计图表（前端渲染）：**
- 练习频率柱状图
- 情绪变化趋势折线图
- 工具使用分布饼图
- 平均专注度变化趋势

**AI 分析洞察：**
- 冥想习惯分析："本周你每天早上 7 点左右进行冥想，这是一个很好的习惯。"
- 情绪趋势解读："你的情绪整体呈上升趋势，特别是在呼吸练习后情绪提升明显。"
- 个性化建议："建议尝试将冥想时间从 10 分钟延长到 15 分钟，你已经具备了这个能力。"
- 工具推荐："你很少使用身体扫描，这可能对缓解你提到的肩部紧绷有帮助。"

**里程碑庆祝：**
- 连续打卡天数
- 总冥想时长里程碑（1h/5h/10h/50h/100h）
- 日记记录条数里程碑

#### 2.3.3 多模型支持

**AI 适配层架构：**

```typescript
interface AIProvider {
  name: string;
  generateSummary(data: MeditationData[]): Promise<SummaryResult>;
}

// 支持的模型
- OpenAI: GPT-4o-mini / GPT-4o
- 通义千问: Qwen
- 可扩展接入更多模型
```

**用户配置：**
- 在设置页面选择 AI 服务商
- 输入自己的 API Key
- API Key 加密存储在 Firebase
- 前端通过 Firebase Cloud Functions 代理调用，不暴露 Key

#### 2.3.4 技术实现

- Firebase Cloud Functions 定时触发（周报/月报）
- Cloud Functions 作为代理层调用 AI API
- 生成结果存入 Firestore `users/{userId}/insights/{insightId}`
- 前端从 Firestore 读取并渲染

### 2.5 模块五：社区 — 共同冥想房间 `新增`

#### 2.5.1 房间机制

**创建房间：**
- 选择冥想类型（自由冥想/呼吸练习/引导冥想）
- 设置时长（5/10/15/20/30 分钟）
- 选择公开/私密
- 公开房间出现在房间列表
- 私密房间通过邀请链接加入

**房间限制：**
- 最多 20 人
- 创建者为房主，控制开始/结束

#### 2.5.2 体验流程

```
进入房间 → 等待阶段（可看到在线用户头像气泡）
  → 房主点击"开始" → 全员同步倒计时
  → 冥想进行中（显示参与人数、剩余时间）
  → 时间到，结束提示音
  → 各自跳转到日记记录页面
```

**关键设计：**
- **不需要音视频通话**，只是"知道有人陪你一起"的陪伴感
- 冥想中仅显示最小化 UI：参与人数 + 倒计时
- 柔和的动画背景（可选可视化场景）

#### 2.5.3 技术实现

- **Firebase Realtime Database** 做在线状态和倒计时同步
- 不需要 WebRTC，轻量级实现
- 房间状态：waiting → meditating → completed
- 自动清理超过 2 小时的空房间

**数据结构：**

```
rooms/{roomId}
  - hostId: string
  - type: string
  - duration: number
  - isPublic: boolean
  - status: 'waiting' | 'meditating' | 'completed'
  - startTime: number | null
  - participants: {
      [userId]: {
        displayName: string
        avatarUrl: string
        joinedAt: number
      }
    }
  - createdAt: number
```

### 2.6 模块六：社区 — 许愿树 `新增`

#### 2.6.1 视觉设计

- 一棵**可视化的大树**（SVG 或 Canvas 渲染）
- 树有轻微的**随风摇摆动画**
- 树的规模随社区叶子总数增长：
  - 0-50 片叶子：小树苗
  - 50-200 片：小树
  - 200-500 片：中等大树
  - 500+ 片：参天大树
- 暗黑模式下树有萤火虫点缀效果

#### 2.6.2 叶子交互

**挂叶子：**
- 输入一句冥想感悟/祝福/鼓励（限 100 字）
- 叶子颜色根据内容情绪自动分配：
  - 暖色系（金黄、橙红）= 喜悦、感恩
  - 冷色系（蓝绿、靛蓝）= 平静、宁静
  - 中性色（翠绿）= 日常感悟
- 新挂的叶子有**飘落挂上**的动画效果
- 每人每天最多挂 **3 片叶子**（防刷屏）

**互动：**
- 点击叶子展开阅读完整内容
- 可以给别人的叶子**浇水**（点赞的诗意表达）
- 被浇水多的叶子会微微发光

#### 2.6.3 数据结构

```
wishTree/leaves/{leafId}
  - userId: string
  - displayName: string
  - content: string
  - color: string
  - waterCount: number  // 浇水数
  - position: { x: number, y: number }  // 在树上的位置
  - createdAt: Timestamp
```

### 2.10 模块十：课程与挑战计划 `新增`

#### 2.10.1 预设课程

| 课程名称 | 天数 | 每日内容 | 难度 |
|---------|------|---------|------|
| 冥想入门 7 天 | 7 | 阅读 + 5分钟练习 | 初级 |
| 21 天正念挑战 | 21 | 阅读 + 10分钟练习 + 日记 | 中级 |
| 30 天呼吸训练 | 30 | 呼吸技法学习 + 练习 | 中级 |
| 14 天身心放松 | 14 | 身体扫描 + 放松练习 | 初级 |

#### 2.10.2 课程机制

- 每天解锁一个练习任务 + 推荐阅读
- 完成当天任务后打勾标记
- 允许补做（最多落后 3 天）
- 完成整个课程获得**成就徽章**
- 课程页面显示整体进度环

#### 2.10.3 数据结构

```
users/{userId}/challenges/{challengeId}
  - courseId: string
  - startDate: Timestamp
  - currentDay: number
  - completedDays: number[]
  - status: 'active' | 'completed' | 'abandoned'
  - badge: string | null
  - createdAt: Timestamp
```

## 5. 非功能性需求

### 5.1 性能

- 首屏加载 < 2 秒（Lighthouse Performance > 90）
- 工具页面动画流畅（60fps）
- 可视化冥想场景使用 Canvas/WebGL，不影响主线程

### 5.2 安全

- API Key 加密存储，通过 Cloud Functions 代理调用
- Firestore 安全规则：用户只能读写自己的数据
- 公开数据（社区、许愿树）有独立的读取规则
- 输入内容 XSS 防护

### 5.3 可访问性

- WCAG AA 标准
- 颜色对比度 ≥ 4.5:1
- 键盘导航支持
- 屏幕阅读器兼容

### 5.4 响应式设计

- Mobile First
- 断点：sm(640px) / md(768px) / lg(1024px) / xl(1280px)
- 工具页面在移动端需特别优化交互体验
